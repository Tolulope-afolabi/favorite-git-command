My Favorite Git command

git push

What does it do?

The git push command is used to upload local repository content to a remote repository. Pushing is how you transfer commits from your local repository to a remote repo. It's the counterpart to git fetch , but whereas fetching imports commits to local branches, pushing exports commits to remote branches (i.e Send changes to the master branch of your remote repository).

Flags that can be used with "git push"

-u (Upstream)

Since the Purpose of git push command is to update our remote repo, The -u flag is used to set origin as the upstream remote in your git config. As you push a branch successfully or update it, it adds upstream reference. As you push local branch with git push -u option, that local branch is linked with the remote branch automatically.

"Upstream" would refer to the main repo that other people will be pulling from, e.g. your GitHub repo. The -u option automatically sets that upstream for you, linking your repo to a central one. That way, in the future, Git "knows" where you want to push to and where you want to pull from, so you can use git pull or git push without arguments. A little bit down, this article explains and demonstrates this concept.

-f (Force)

git push -f will force push all of the tracking branches to remote. If a developer has a few outdated branches, that are tracking remotes, and he executes that command, all of the tracking branches will have been rolled back to his outdated copies, which causes loss of valuable work.

This can be done accidentally, or by someone not very experienced with git like myself (haha.). It really seems like git should ask for a confirmation just so we do not use this unintended (like adding an extra check in the -f behavior say "Are you sure you want to delete the file?" and then "Are you really sure?").
